# 2026-04-03 — MAX интеграция и аудит мини-аппа

## Сделано

- **CSS полностью переписан** — был CSS от beauty studio (розовый #e91e8c), заменён на keychain shop стили (Telegram-синий #0088cc, все классы из index.html)
- **Добавлены ~20 отсутствующих CSS-классов**: `.quantity-btn`, `.form-section`, `.checkout-summary`, `.success-order-id`, `.empty-cart`, `.empty-orders` и др.
- **Исправлен баг с маской телефона**: `initPhoneMask()` вызывался при старте до рендера формы — перенесён в конец `renderCheckout()`
- **Убран двойной вызов `telegramApp.init()`** (конструктор уже вызывает его)
- **Заменены сломанные Unsplash-картинки** (все 404) на эмодзи для каждого из 15 товаров
- **Переход на MAX WebApp SDK**: `window.WebApp` (MAX) → `window.Telegram.WebApp` (Telegram) → mock; оба SDK подключены в index.html
- **Кнопка «Поделиться»**: добавлена на экран успешного заказа, вызывает `shareMaxContent()` по документации MAX
- **`contact_by` перебрал несколько значений**: phone → id → max_id → telegram_id. Итог: `telegram_id` — MAX user ID хранится в этом поле LeadTeX
- **Убраны все alert()** — в MAX не работают; валидация формы оставляет только `focus()` на поле
- **Исправлен `requestContact`**: неверное событие `contactRequested` → `WebAppRequestPhone`, поле `data.contact.phone_number` → `data.phone` (по документации MAX)
- **Кнопка «Получить из MAX»** в итоге удалена — нестабильна, телефон не нужен для поиска контакта
- **jest testMatch** ограничен `app.test.js` — `config.test.js` и `cart.test.js` были orphaned

## Решения принятые сегодня

- Поиск контакта в LeadTeX: `contact_by: 'telegram_id'`, `search: MAX user ID` — LeadTeX не имеет поля max_id, хранит внешний ID мессенджера в telegram_id
- Картинки товаров: эмодзи надёжнее внешних URL в Telegram Mini App
- Алерты убраны полностью, без замены — они ломают UX в MAX
- Кнопка шеринга только на экране успешного заказа, только в MAX (`isMAX` флаг)
