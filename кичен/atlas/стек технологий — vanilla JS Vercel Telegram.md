# Стек технологий — vanilla JS, Vercel, Telegram

## Фронтенд

| Технология | Использование |
|---|---|
| HTML5 | Единственный `index.html`, 6 секций-экранов |
| Vanilla JavaScript | Вся логика без фреймворков |
| CSS3 | Grid, Flexbox, CSS Variables, темы dark/light |
| Telegram Web App SDK | `window.Telegram.WebApp` — нативный SDK |

**Намеренный отказ от фреймворков**: нет React, Vue, Angular. Приложение небольшое, статичное, деплоится без сборки. Добавление бандлера усложнило бы деплой без реальной выгоды.

## Хостинг и бэкенд

| Технология | Использование |
|---|---|
| Vercel | Хостинг статики + Serverless Functions |
| Node.js (serverless) | `api/webhook.js` — CORS прокси |
| GitHub | Источник для auto-deploy |

## Тестирование

| Технология | Использование |
|---|---|
| Jest 29 | Фреймворк тестирования |
| jest-environment-jsdom | Эмуляция браузера в Node.js |

Тесты в `tests/` покрывают: корзину, каталог, навигацию, отправку заказа.

## Внешние сервисы

| Сервис | Роль |
|---|---|
| LEADTEX (leadteh.ru) | CRM: контакты, сценарии, уведомления менеджерам |
| Telegram Bot API | Регистрация бота, настройка Mini App кнопки |
| Unsplash CDN | Изображения товаров (не хостятся локально) |

## Нет в стеке (и почему)

- **База данных** — заказы хранятся в LEADTEX, корзина в `localStorage`. [[нет базы данных — состояние хранится в памяти и LEADTEX]]
- **Аутентификация** — роль "пользователя" выполняет Telegram ID, никакой отдельной авторизации
- **Платёжная система** — заказ отправляется в CRM, оплата вне приложения
