# Нет базы данных — состояние хранится в памяти и External CRM

## Где что хранится

| Данные | Где хранятся | Как долго |
|---|---|---|
| Корзина (объекты товаров) | `cart[]` в памяти JS | До перезагрузки страницы |
| `telegram_id` пользователя | `localStorage` | Постоянно (до очистки браузера) |
| Данные контакта | External CRM CRM | Постоянно |
| История заказов | External CRM CRM | Постоянно |
| Каталог товаров | `js/config.js` (статика) | Деплой |

## Следствия этой архитектуры

**Плюсы:**
- Нет своего сервера, нет своей БД — нет расходов и поддержки
- Деплой = пуш в GitHub

**Минусы:**
- Корзина сбрасывается при закрытии вкладки (нет сохранения в `localStorage`)
- Секция «Мои заказы» — заглушка, реальной истории нет без API к External CRM
- Нельзя показать статус заказа внутри Mini App без обратного вызова от External CRM

## telegram_id как идентификатор пользователя

Приложение не имеет собственной системы пользователей. Вместо этого:

1. Telegram передаёт `id` через `window.Telegram.WebApp.initDataUnsafe.user`
2. `telegram.js` сохраняет его в `localStorage` при инициализации
3. `app.js::sendOrderToServer()` читает из `localStorage` с fallback на `window.Telegram.WebApp`

Это означает: **пользователь должен сначала написать `/start` боту**, чтобы External CRM создал для него контакт. Если контакта нет — поиск по `telegram_id` вернёт пустой результат.
