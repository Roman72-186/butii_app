# Архитектура системы построена на трёх уровнях

## Уровни системы

```
┌──────────────────────────────────────────────┐
│  УРОВЕНЬ 1: Telegram Mini App (фронтенд)      │
│                                              │
│  index.html  →  js/config.js                 │
│                 js/telegram.js               │
│                 js/app.js                    │
│                 css/style.css                │
└──────────────────────────────────────────────┘
                       │ POST /api/webhook
                       ▼
┌──────────────────────────────────────────────┐
│  УРОВЕНЬ 2: Vercel Serverless (прокси)        │
│                                              │
│  api/webhook.js  — решает проблему CORS      │
│  vercel.json     — маршруты + заголовки      │
└──────────────────────────────────────────────┘
                       │ POST inner_webhook
                       ▼
┌──────────────────────────────────────────────┐
│  УРОВЕНЬ 3: LEADTEX CRM (бэкенд)             │
│                                              │
│  Хранит контакты, запускает сценарии,        │
│  уведомляет менеджеров                       │
└──────────────────────────────────────────────┘
```

## Почему три уровня, а не два

Telegram Mini App не может напрямую обратиться к LEADTEX из-за ограничений CORS. Браузер внутри Telegram блокирует cross-origin запросы к чужому домену. Vercel Serverless Function работает как сервер — для неё CORS не существует. Поэтому все запросы идут через `/api/webhook`.

Подробнее: [[выбор Vercel как CORS прокси вместо собственного бэкенда]]

## Отдельный компонент — Bridge

Для рекламных кампаний существует отдельный Mini App в `bridge/`. Он решает другую проблему — потерю UTM-параметров для новых пользователей. Архитектурно повторяет ту же схему: свой `vercel.json` + своя serverless функция `api/bridge-webhook.js`.
