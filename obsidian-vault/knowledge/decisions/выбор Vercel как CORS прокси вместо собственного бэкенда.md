# Выбор Vercel как CORS прокси вместо собственного бэкенда

## Контекст

Telegram Mini App запускается в WebView внутри Telegram. Браузер блокирует запросы с одного домена (`vercel.app`) к другому (`leadteh.ru`) из-за политики Same-Origin. Прямой вызов LEADTEX webhook из фронтенда невозможен.

## Варианты решения

| Вариант | Плюсы | Минусы |
|---|---|---|
| Vercel Serverless Function (выбрано) | Нет своего сервера, бесплатно, деплой = пуш | Ограничения cold start, нет полного контроля |
| Свой Node.js/Python сервер | Полный контроль | Нужен VPS, деплой, мониторинг, расходы |
| CORS заголовки на стороне LEADTEX | Простейшее решение | LEADTEX не поддерживает кастомные CORS headers |
| Vercel Rewrite (без кода) | Ещё проще | Нет обработки ошибок, нет логирования |

## Решение

`api/webhook.js` — serverless функция, которая:
1. Принимает POST от фронтенда
2. Делает `fetch` к LEADTEX (server-to-server, CORS не применяется)
3. Возвращает ответ фронтенду

Одновременно в `vercel.json` есть `rewrites` на тот же URL — это дублирование. Serverless-функция предпочтительнее: она добавляет логирование и обработку ошибок.

## Текущий статус

Оба механизма (`api/webhook.js` и `vercel.json → rewrites`) указывают на один LEADTEX URL. При обновлении UUID вебхука нужно менять оба файла. Это технический долг.
